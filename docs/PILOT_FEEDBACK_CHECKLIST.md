# Partlink Pilot Feedback Checklist

Έκδοση: 1.0 — 2026-05-14

---

## 1. Στόχος feedback

Αυτό το checklist χρησιμοποιείται κατά τη διάρκεια ή αμέσως μετά την παρουσίαση του Partlink MVP σε πιλοτικό χρήστη (πωλητή ή συνεργείο).

**Τι θέλουμε να επικυρώσουμε:**

| Ερώτημα | Τι ελέγχουμε |
|---------|--------------|
| Seller workflow | Ο πωλητής καταλαβαίνει πώς να ανεβάσει ανταλλακτικά; Θα το έκανε; |
| Request-first flow | Ο αγοραστής βρίσκει λογικό να στέλνει αίτημα αντί να αγοράζει απευθείας; |
| VIN import | Ο πωλητής καταλαβαίνει τη ροή εισαγωγής αυτοκινήτου με VIN; Θα τη χρησιμοποιούσε; |
| QR scan | Το QR label φαίνεται χρήσιμο για τη μάντρα ή φαίνεται άχρηστη επιπλοκή; |
| Admin επαλήθευση | Η χειροκίνητη επαλήθευση πωλητή φαίνεται λογική; Δημιουργεί εμπιστοσύνη; |
| Εμπόδια καθημερινής χρήσης | Τι θα τον εμπόδιζε να χρησιμοποιεί το Partlink κάθε μέρα; |

---

## 2. Πριν το demo

Βεβαιωθείτε ότι τα παρακάτω είναι έτοιμα:

- [ ] Η εφαρμογή τρέχει τοπικά: `npm run dev` στο `http://localhost:3000`
- [ ] Έχετε ανοιχτές τις βασικές σελίδες σε tabs: `/`, `/marketplace`, `/seller`, `/seller/inventory/add`, `/seller/inventory/vin-import`, `/seller/orders`, `/buyer/orders`, `/admin`
- [ ] Έχετε εξηγήσει ότι το demo είναι **τοπικό πρωτότυπο** — δεν υπάρχει real backend, auth ή δεδομένα που αποθηκεύονται μετά από refresh
- [ ] Έχετε επιλέξει ποιο mock αίτημα θα χρησιμοποιήσετε για τη ροή (προτείνεται: req-003 για πωλητή, req-004 για αγοραστή)
- [ ] Έχετε ανοιχτό αρχείο σημειώσεων (φυσικό ή ψηφιακό) για να καταγράφετε αντιδράσεις σε πραγματικό χρόνο
- [ ] Έχετε διαβάσει τις ερωτήσεις του §5 και §6 πριν ξεκινήσετε

## 3. Κατά τη διάρκεια του demo — Παρατηρήσεις

Σημειώστε ό,τι παρατηρείτε χωρίς να κατευθύνετε τον χρήστη.

### 3.1 Αντίδραση πωλητή

- [ ] Ο πωλητής καταλαβαίνει αμέσως τι κάνει ο seller dashboard;
- [ ] Ρωτά για τον τρόπο ανεβάσματος ανταλλακτικών χωρίς να του εξηγηθεί;
- [ ] Κοιτάει το inventory list και σχολιάζει ότι μοιάζει με αυτό που έχει τώρα;
- [ ] Σχολιάζει θετικά ή αρνητικά τα KPI cards (stock, αιτήματα, πωλήσεις);
- [ ] Ρωτά για τον τρόπο πρόσβασης (mobile / desktop);

### 3.2 Αντίδραση στη ροή αγοραστή

- [ ] Ο πωλητής ή ο buyer βρίσκει φυσική τη ροή: marketplace → part detail → αίτημα;
- [ ] Η έννοια του αιτήματος (αντί για άμεση αγορά) φαίνεται κατανοητή;
- [ ] Ο αγοραστής καταλαβαίνει ότι τα στοιχεία του φορτώνονται αυτόματα από το προφίλ;
- [ ] Ρωτά τι γίνεται μετά την αποδοχή τιμής;

### 3.3 Αντίδραση στο inventory

- [ ] Ο πωλητής καταλαβαίνει ότι κάθε ανταλλακτικό έχει SKU;
- [ ] Τον ενδιαφέρουν τα φίλτρα (κατηγορία, κατάσταση, marketplace visibility);
- [ ] Σχολιάζει ότι λείπει κάτι από τα στοιχεία του ανταλλακτικού;
- [ ] Ρωτά για import από Excel ή άλλο σύστημα που χρησιμοποιεί τώρα;

### 3.4 Αντίδραση στο VIN import

- [ ] Καταλαβαίνει τη ροή: VIN → αποκωδικοποίηση → λίστα ανταλλακτικών → bulk publish;
- [ ] Σχολιάζει ότι η αποκωδικοποίηση είναι mock και ρωτά αν θα λειτουργεί στο production;
- [ ] Φαίνεται να καταλαβαίνει την αξία: «φέρνω αυτοκίνητο, βγάζω λίστα αμέσως»;
- [ ] Ρωτά για συμβατότητα με συγκεκριμένα μοντέλα που έχει;

### 3.5 Αντίδραση στο QR scan

- [ ] Καταλαβαίνει γιατί υπάρχει το QR label;
- [ ] Σχολιάζει θετικά τη σύνδεση QR → ανταλλακτικό → stock update;
- [ ] Φαίνεται ότι ήδη χρησιμοποιεί κάποια μορφή labeling / ετικετών;
- [ ] Ρωτά για τον εκτυπωτή ετικετών;

### 3.6 Αντίδραση στην επαλήθευση / admin

- [ ] Βρίσκει λογικό ότι χρειάζεται manual approval για να πουλά;
- [ ] Ρωτά πόσο χρόνο παίρνει η επαλήθευση;
- [ ] Ο αγοραστής σχολιάζει ότι οι πωλητές φαίνονται αξιόπιστοι λόγω επαλήθευσης;

## 4. Ερωτήσεις για πιλοτικό πωλητή (μάντρα)

Ρωτήστε ανοιχτά — μην κατευθύνετε την απάντηση.

**Ροή ανεβάσματος ανταλλακτικών:**

1. Θα ανεβάζατε ανταλλακτικά με αυτόν τον τρόπο;
   Είναι εύκολο; Τι βήμα θα αλλάζατε;

2. Ποιο βήμα σας φαίνεται αργό ή περιττό;

3. Πόσα ανταλλακτικά έχετε τώρα; Πώς τα διαχειρίζεστε σήμερα;

**Αιτήματα αγοραστών:**

4. Τι πληροφορίες χρειάζεστε πριν στείλετε τιμή σε έναν αγοραστή;
   Λείπει κάτι από αυτό που βλέπετε στο αίτημα;

5. Προτιμάτε ο αγοραστής να στέλνει αίτημα ή να μπορεί να αγοράζει απευθείας;

**QR labels και αποστολή:**

6. Θα βοηθούσαν τα QR labels στη μάντρα; Πώς ελέγχετε τώρα τι φεύγει;

7. Έχετε αντικλεφτικό ή labeling σύστημα τώρα; Τι χρησιμοποιείτε;

**Οργάνωση:**

8. Ποιος στη μάντρα θα χρησιμοποιούσε αυτό; Μόνοι σας ή με υπαλλήλους;

9. Τι θα σας εμπόδιζε να το χρησιμοποιείτε κάθε μέρα;
   Χρόνος; Εκπαίδευση; Τεχνολογία; Τιμή;

---

## 5. Ερωτήσεις για συνεργείο / αγοραστή

**Αναζήτηση:**

1. Θα ψάχνατε ανταλλακτικά έτσι αντί να καλείτε μάντρες μία-μία;

2. Πόσες μάντρες συνήθως καλείτε για να βρείτε ένα ανταλλακτικό;

**Request-first ροή:**

3. Καταλαβαίνετε τη ροή; Στέλνετε αίτημα → ο πωλητής απαντά → αποδέχεστε τιμή.
   Ή περιμένετε να δείτε τιμή αμέσως;

4. Τι χρειάζεστε πριν αποδεχτείτε τιμή;
   Φωτογραφία; Εγγύηση; Πληροφορία για κατάσταση; Κάτι άλλο;

**VIN αναζήτηση:**

5. Αν γνωρίζετε το VIN, θα το χρησιμοποιούσατε για αναζήτηση;

**Παράδοση:**

6. Είναι κατανοητή η επιλογή αποστολής / παραλαβής; Τι προτιμάτε συνήθως;

**Εμπιστοσύνη:**

7. Τι θα σας έκανε να εμπιστευτείτε έναν πωλητή στο Partlink;
   Επαλήθευση; Αξιολογήσεις; Φωτογραφίες; Κάτι άλλο;

## 6. Σήματα κινδύνου

Αν παρατηρήσετε τα παρακάτω, σημειώστε τα αμέσως. Χρειάζονται προσοχή.

| Σήμα | Τι σημαίνει |
|------|-------------|
| Ο πωλητής δεν θέλει να ανεβάζει stock ψηφιακά | Η κεντρική αξία δεν πείθει — χρειάζεται επανεξέταση positioning ή UX |
| Το VIN import φαίνεται μπερδεμένο | Η ροή χρειάζεται απλοποίηση ή καλύτερη εξήγηση |
| Τα QR labels φαίνονται σαν επιπλέον δουλειά | Η αξία του QR δεν είναι ξεκάθαρη — χρειάζεται καλύτερη παρουσίαση |
| Ο αγοραστής θέλει μόνο τηλεφωνική επικοινωνία | Η request-first ροή δεν αντικαθιστά ακόμα το τηλέφωνο στο μυαλό του |
| Η request-first ροή φαίνεται αργή ή περιττή | Η διαδικασία ανταλλαγής τιμής δεν φαίνεται ρεαλιστική |
| Η επαλήθευση φαίνεται εμπόδιο χωρίς αξία | Το trust layer δεν εξηγείται καλά |
| Ο χρήστης ρωτά για το ίδιο πράγμα πολλές φορές | UX ή copy δεν είναι κατανοητό |
| Κανείς δεν σχολιάζει τον πίνακα ελέγχου θετικά | Το dashboard δεν δείχνει αξία |

## 7. Ισχυρά θετικά σήματα

Αν δείτε τα παρακάτω, είναι καλό σημάδι. Καταγράψτε τα.

| Σήμα | Τι σημαίνει |
|------|-------------|
| Ο πωλητής ρωτά πώς να ανεβάσει πραγματικό stock | Η κεντρική αξία πείθει |
| Ο πωλητής ρωτά για εκτυπωτή QR labels | Ο QR dispatch γίνεται κατανοητός και χρήσιμος |
| Ο αγοραστής λέει «αυτό εξοικονομεί κλήσεις» | Το request-first αντικαθιστά ρεαλιστικά τη σημερινή ροή |
| Ο pilot ρωτά για real account ή backend | Είναι έτοιμος για επόμενο βήμα |
| Ο πωλητής ρωτά αν μπορεί να δώσει πρόσβαση σε υπάλληλο | Βλέπει real operational χρήση |
| Κάποιος ρωτά για τιμολόγιο / παραστατικό | Βλέπει real transaction αξία |
| Κάποιος ρωτά «πότε ανοίγει;» | Υπάρχει πραγματικό ενδιαφέρον |
| Ο pilot δίνει επαφή άλλης μάντρας να το δει | Organic word-of-mouth |
